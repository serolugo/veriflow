feat(docker): replace GTKWave/noVNC with Surfer WASM viewer

Motivation
----------
TileBench was updated to drop the X11/VNC stack (Xvfb, x11vnc, noVNC)
in favour of Surfer WASM served locally. VeriFlow's Docker-aware wave
launching needed to match — GTKWave no longer exists in the container.

Changes
-------
sim_runner.py
  - Remove _launch_gtkwave_docker() — no longer valid without X11
  - Add open_surfer(wave_path): public function for Docker wave viewing
      * Resolves VCD path relative to /workspace
      * Constructs Surfer URL with ?load_url= for VCD auto-load
      * Prints direct URL to terminal for copy-paste
      * Calls webbrowser.open() as best-effort (may not open on host)
      * Gracefully handles VCDs outside /workspace
  - launch_gtkwave() Docker branch now delegates to open_surfer()

waves.py
  - cmd_waves() Docker branch now calls open_surfer() directly
  - Removes subprocess + DISPLAY :1 + BROWSER_OPEN env var logic

Non-Docker behaviour (GTKWave on Linux/Mac/Windows) is unchanged.
